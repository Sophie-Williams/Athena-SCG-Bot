Distribution

You should have recived the Starting Player as a JAR (Java ARchive) or a Tar/GZip file. It should include all the source, shell scripts and other files needed to get you started. The only difference between the two that the TarGZip file includes the 'Player' base directory.

The JAR is runnable, and you can test it with: 

    java -cp $CLASSPATH:player.jar player.PlayerMain
If you get errors, then most likely the DemeterF JAR is not correctly in your classpath. The Tar/GZip file can be unpacked into your Eclipse workspace, and imported.

Files and Organization

The starting player is organized as an Eclipse project. It can be unpacked into a workspace directory (TarGZip version contains it's own base directory). The directories are organized as follows: 

    Player/
       PlayerTest.java
       makeJar.sh
       runPlayer.sh

       files/
          context.txt
    
       player/
          PlayerMain.java
          Register.java
          Player.java
          player.mf
          PlayerServer.java
        
          tasks/
             SolveTask.java
             ReofferTask.java
             AcceptTask.java
             OfferTask.java
             ProvideTask.java

  • Player/: Base directory, contains all the sources. Feel free to rename this for your player.
  • PlayerTest.java: A simple Test class for the Player.
  • makeJar.sh: Script used to make the JAR distribution.
  • makeTar.sh: Script used to make the Tar/GZip distribution.
  • files/: Use this to put any test (non-java) files or other configuration files for your player. Currently there's only one file, used to test the Player with a simple context.
  • context.txt: Contains a simple PlayerContext to test the Player classes. See PlayerTest.java for how we use it.
  • player/: The player package, for all things Player related.
  • PlayerMain.java: The Main class for running the Player; everything starts here.
  • Register.java: Competitions require registration with the Administrator. This class handles that, and can be used/run to register a Player seperately for testing or multiple competitions.
  • Player.java: Deligation point for the Player tasks. Created with a PlayerContext and a Logger. When play() is called it inturn calls each of the player tasks, to create a final PlayerTransaction.
  • player.mf: A JAR manifest file for the Player. This is included in the JAR to make it runnable, though there needs to be a good amount of CLASSPATH setup before it will work with java -jar player.jar
  • PlayerServer.java: The HTTP server portion of the Player. This class handles the setup/binding of the server, and interaction with the Administrator over sockets.
  • tasks/: Package that contains the Player's tasks/functions.
  • SolveTask.java: A basic Solution task. Basically creates a random assignment for the variables of the given Problem.
  • ReofferTask.java: Reoffers all the Challenges from others. This is used when the Player doesn't find challenges worth accepting.
  • AcceptTask.java: Figure out what challenges to accept. This is currently done randomly, but you should add more intelligence.
  • OfferTask.java: Creates new challenges to be offered. Currently random, but you should offer challenges based on more information.
  • ProvideTask.java: Provides Problems (once accepted) for others to solve.
